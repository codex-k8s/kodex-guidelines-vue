# Инструкции для агентов

## Назначение репозитория

Этот репозиторий содержит инженерные правила для frontend-стека Vue 3 + TypeScript + Pinia + Axios + Vite.

## Правила работы

- Ответы пользователю и оформление PR писать на русском языке.
- Тексты commit message писать на английском языке.
- Прямой push в `main` запрещён: изменения вносятся через отдельную ветку и PR.
- Если этот репозиторий используется как внешний пакет с локальными копиями в проектах-потребителях, проверить, нужно ли синхронизировать эти копии.
- Репозиторий должен оставаться самодостаточным: не добавлять обязательных ссылок на другие пакеты правил.
- Не добавлять обязательные нормы, завязанные на конкретный проект, продукт, slug репозитория, схему входа, naming conventions или частную delivery-политику.
- Проектные overlays должны жить в локальной документации проекта-потребителя.
- Примеры допускаются, если они явно помечены как примеры и не превращаются в обязательную норму для всех потребителей.

## Состав репозитория

Документы для frontend (Vue 3 + TypeScript).

- `check_list.md` — чек-лист перед PR для Vue изменений.
- `frontend_architecture.md` — размещение, структура приложения, границы ответственности.
- `frontend_data_and_state.md` — axios/Pinia/router/i18n/cookies/PWA/WebSocket правила.
- `frontend_error_handling.md` — модель и обработка ошибок на фронте.
- `frontend_code_rules.md` — правила кодирования (TS/Vue/импорты/комментарии).
- `libraries.md` — что выносить в `libs/{vue,ts,js}` и как.

Конкретные project-level ограничения, auth/SSO choices и UX-conventions должны задаваться локальным overlay проекта-потребителя.
